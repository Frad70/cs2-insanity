# INSANITY REVIVE

> _«Когда-нибудь ты не отличишь их от настоящих игроков. И вот тогда станет по-настоящему страшно.»_

---

## О проекте

**INSANITY REVIVE** — это плагин для **Counter-Strike 2**, написанный на платформе
[CounterStrikeSharp (CSSharp)](https://docs.cssharp.dev/) под .NET 8.
Цель проекта одна и сформулирована намеренно резко: сделать ботов **неотличимыми
от живых матчмейкинговых игроков** — по чату, по движению, по аиму, по решениям,
по выбору целей, по командной работе.

Это не «улучшенные боты». Это попытка построить поверх стандартного
бот-движка CS2 целый слой поведения — со своими личностями, настроениями,
коллбэками по карте, экономической моделью, клатч-поведением и реакцией
на события матча. Каждый бот — маленький автомат, у которого есть стиль
письма, темп печати, любимое оружие, нервы, наклон в сторону тильта или
уверенности, и привычка разговаривать с командой.

Сам по себе CS2 даёт ботов, которые ведут себя как боты. **INSANITY REVIVE**
делает так, чтобы вы перестали быть в этом уверены.

## Что внутри (общими словами)

Плагин — один C#-проект, единое пространство имён `InsanityRevive`,
работает поверх `CounterStrikeSharp.API`. Архитектурно это **один
host-плагин** и набор **сервис-классов**, каждый из которых отвечает
за свою грань поведения бота.

### Модули поведения

- **Aim Controller** — плавное предсказательное прицеливание.
  У каждого бота свой профиль аима: скорость доводки, биас, шум руки,
  микрокоррекции, флики, спрей, время реакции. Это не классический
  «aimbot one-tap», а имитация того, как целится живой человек —
  с дрожью, с небольшим перелётом, с задержкой на распознавание цели.

- **Decision Engine** — настроение, уверенность, серии килов и смертей,
  «чад-режим» после хороших раундов, тильт после плохих. Решения
  бота не статичны: один и тот же бот в один и тот же момент карты
  может сыграть по-разному в зависимости от того, как у него идёт матч.

- **Clutch Behavior** — отдельная подсистема для ситуаций 1vN.
  Замедление темпа, тишина в чате, аккуратные перемещения — то,
  как клатчат живые игроки, а не то, как «доигрывает раунд» обычный бот.

- **Economy Model** — командная закупочная логика. Боты считают деньги,
  смотрят на экономику противника, договариваются об эко/форс/фул-баях
  на уровне команды, а не каждый сам по себе.

- **Buy Preferences** — личные предпочтения по оружию у каждого бота.
  Кто-то любит AWP, кто-то всегда берёт SCAR, кто-то экономит и сидит
  на дефолтном пистолете до последнего.

- **Map Knowledge** — знания о картах: имена зон для коллаутов,
  наличие AWP-углов, времена ротаций, типичные позиции.
  Без этого никакие «голосовые» сообщения в чате не звучат естественно.

- **Movement Realism** — реалистичные движения: контр-страйфы,
  пиковые ритмы, приседания, прыжки, мелкие колебания скорости.
  Принципиально **только через кнопки**, без вмешательства в скорость
  напрямую — иначе ботов мгновенно выдаёт неестественный дрейф.

- **Chat Styles + Bot Persona** — у каждого бота своя личность:
  стиль письма, словарь, скорость печати в WPM, склонность к токсику
  или дружелюбию, типажи поведения. Чат проходит через пайплайн
  с задержкой на «печать», случайными опечатками, шансом отправить
  «не туда» (team_only → all) и цепочками рибуков от тиммейтов.

### Темп работы

Сердце плагина — таймер на **33 Гц**, который дёргает все подсистемы:
аим, кнопки движения, реализм, форс-взгляд, флаш логов поведения.
Это горячий путь, и он вылизан так, чтобы переживать большое
количество ботов на сервере без аллокаций в каждом тике.

### Личности и пресеты сложности

Поверх всего этого — **пресеты сложности**:
`casual`, `normal`, `hard`, `insane`, `aimbot`. Каждый пресет
переключает не только параметры аима, но и сам файл `botprofile.vpk`,
который читает движок CS2. Смена пресета аккуратно перезагружает
текущую карту, чтобы профиль применился штатно.

## Установка

> **Установка возможна только для владельца проекта и участников из
> списка _Contributors_.** Если вы не из этого списка — пропустите
> этот раздел. Серьёзно. Пожалуйста.

Для авторизованных участников процесс выглядит так (общими словами,
без публичной инструкции):

1. Сборка проекта в `Release` через стандартный `dotnet build`.
2. Копирование собранной DLL и PDB в директорию плагинов CSSharp
   на целевом сервере CS2.
3. Применение нужных бот-профилей (`botprofile.vpk`) из набора
   `overrides/`, соответствующих выбранному пресету сложности.
4. Перезагрузка плагина внутри игры через стандартную команду
   CSSharp на горячую перезагрузку плагинов.
5. Проверка поведения — не по subjective feel, а по структурированному
   логу поведения, который пишет сам плагин.

Тонкие детали путей, конфигов, прав и интеграции с конкретным сервером —
**вне репозитория**. Это сделано специально, чтобы случайный человек
не смог собрать рабочий деплой даже при доступе к коду.

## Команды в игре

Плагин регистрирует несколько админских команд (доступны только
обладателям соответствующих прав на сервере):

- **Открыть меню радио и настроек ботов** — ChatMenu для оперативной
  настройки поведения прямо во время игры.
- **Сменить пресет сложности** — выбор одного из заранее
  подготовленных профилей (`casual` / `normal` / `hard` / `insane` /
  `aimbot`). Триггерит корректную перезагрузку карты, чтобы движок
  подхватил новый бот-профиль.
- **Тогглы отдельных подсистем** — например, переопределение
  логики закупа по персоне (по умолчанию выключено, потому что
  конфликтует с встроенной AI-логикой движка).

Конкретные имена команд и формат аргументов специально не вынесены
в публичный README.

## Принципы разработки

В коде есть несколько правил, которым проект следует жёстко:

1. **Никаких прямых записей в скорость pawn'а.**
   Только кнопки и углы взгляда. Любая попытка двигать ботов через
   `AbsVelocity` уже один раз приводила к багу с дрейфом и
   больше повторяться не должна.

2. **Все per-bot состояния — в словарях по `slot`.**
   Любая новая подсистема, которая хранит per-bot данные, обязана
   очищать их в `Unload` и (если нужно) в начале раунда.

3. **Чат бота никогда не идёт мимо `ScheduleBotChat`.**
   Прямой вывод в чат игнорирует кулдауны, заморозку на время «печати»,
   персону, мангл опечаток и шансы рибука. Поэтому он запрещён.

4. **Дедуп-гейты неприкосновенны.**
   Зональные коллбэки, low-HP колы, кулдауны на чат — всё это нужно,
   чтобы команда не выглядела как спам-бот. Обходить их нельзя.

5. **Roll, а не «всегда делать».**
   Везде, где есть случайное поведение, оно идёт через единый
   `Roll(pct)` (с bot-aware перегрузкой, которая учитывает
   `Talkativeness` персоны), а не через хардкод `Random.Next` по месту.

6. **CSSharp interop оборачивается в try/catch.**
   Один пропавший в середине тика бот не должен класть весь event loop.

7. **Параллельная работа — через partial class.**
   Чтобы несколько веток/агентов могли править плагин одновременно,
   крупные подсистемы выносятся в отдельные `partial class` файлы.

## Версионирование и журнал

- **`ModuleVersion`** в основном файле плагина — единственный источник
  правды о версии. Любая фича бампит её.
- **Коммиты** идут в формате `vX.Y.Z — <однострочное описание>`
  и пушатся в `origin` на каждом нумерованном бампе.
- **Папка `journal/`** — это устойчивый к компакции лог решений,
  итераций и больших архитектурных шагов. Это не «документация
  для пользователей», это рабочая память проекта. Перед любой крупной
  правкой её следует читать.
- В `journal/` есть отдельный протокол автономной сессии
  (`SESSION_START.txt`, `RETURN_PHRASE.txt`) — он рассчитан на
  работу AI-ассистентов в долгих сессиях и не нужен сторонним людям.

## Стек и зависимости

- **Язык:** C#
- **Среда выполнения:** .NET 8
- **Платформа плагинов:** CounterStrikeSharp (минимальная API-версия
  фиксирована в коде плагина)
- **Игра:** Counter-Strike 2
- **Дополнительно:** набор `botprofile.vpk` под каждый пресет
  сложности, лежащий рядом с сервером (вне репозитория).

Тестового проекта нет. Корректность проверяется тремя вещами:
сборка без ошибок, поведение на живом сервере и пост-фактум-анализ
структурированного лога поведения, который плагин пишет сам.

## Заключение

**INSANITY REVIVE** существует для одного: чтобы зайти на
матч с ботами и в какой-то момент поймать себя на мысли
_«стоп, а это вообще боты?»_. Чтобы они переписывались между собой,
ругались на тиммейтов, делали коллауты по зонам, аккуратно клатчили
1vN, по-разному стреляли в зависимости от настроения, и в целом
вели себя так, как ведут себя обычные живые игроки в обычном MM.

Это не продукт. Это эксперимент в близком кругу. И он останется
таким.
