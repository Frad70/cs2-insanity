# Iteration v0.9.0 — zone-aware callouts — 2026-04-30 ~23:45

## Shipped (v0.9.0)

### ChatStyles.cs
- 13 new callout pools: `Callout_Smoke`, `Callout_Molly`, `Callout_Flashed`,
  `Callout_Planted`, `Callout_DefuseCommit`, `Callout_TimeLow`, `Callout_OneShot`,
  `Callout_OneShotZone`, `Callout_ShotsFired`, `Callout_Footsteps`,
  `Callout_Echo`, `Callout_Question`, `Callout_Rebuke`, `Callout_Trade`.
- `ZoneKeyFor(map, line)` — canonicalizes a callout into a "zone family" key
  ("a"/"b"/"mid") for dedup. Longest-match-first (so "site a" beats "a"),
  then regex fallback for raw site/path tokens. Collapses synonyms via
  `CollapseFamily()` — "long a"/"apps"/"banana"/etc → "a".
- `PickCallout(pool, map, who, rng)` and `PickCalloutFixed(pool, zone, who, rng)`
  helpers for {z}/{who} substitution.

### InsanityRevive.cs
- Bumped `ModuleVersion → "0.9.0"`. Updated `ModuleDescription`.
- Registered 6 new event handlers: `EventSmokegrenadeDetonate`,
  `EventInfernoStartburn`, `EventFlashbangDetonate`, `EventPlayerBlind`,
  `EventRoundTimeWarning`, `EventBombBegindefuse`.
- `_zoneCalloutCooldown : Dictionary<string,float>` — global per-zone-key
  cooldown of 7-12s (random per claim). Cleared on round start + Unload.
- `_lowEnemyCallCooldown : Dictionary<int,float>` — per-victim 6s cooldown
  for "one shot {who}" callouts.
- `TryClaimZoneCall(key)` — atomic claim/check helper.
- `ScheduleCalloutChat(bot, line, zoneKey, ...)` — wraps `ScheduleBotChat`,
  adds dedup gate, then on `responsePct` rolls a teammate's persona-driven
  echo/question/rebuke/trade response:
  - **Hostile** + Tab≥Medium → rebuke (toxic)
  - **Friendly** + Tab≥Low → echo or trade
  - **Neutral** / low-tab → question or echo
- `PickIGLOrTalker(team)` — returns alive IGL archetype with 0.7 prob,
  else falls back to `PickWeightedTalker`. Used for `RollRoundStrategy`
  (T-side strat caller) and CT-side "PLANTED A/B" caller.
- `ClosestTeammateBot(team, pos, radius)` — picks alive bot nearest to a
  world position; used for smoke/molly callouts (so the spotter is plausibly
  near the grenade) and footstep callouts.
- Wired into existing triggers:
  - **OnPlayerHurt low-zone callout** (line 590-ish): now passes through
    `TryClaimZoneCall` and `ScheduleCalloutChat` (so it can echo).
  - **OnPlayerHurt low-HP enemy** (NEW): when a bot drops a victim to ≤30 HP,
    fires "one shot {who}" / "low {who}" with per-victim 6s cooldown.
  - **OnWeaponFire / BroadcastShotToNearbyAllies** (NEW): 10% chance the
    closest opp-team bot calls "shots {z}".
  - **OnPlayerFootstep** (NEW): 4% chance opp-team bot calls "steps {z}"
    (dedup keeps this from spamming).
  - **OnBombPlanted** (NEW): CT IGL/talker calls "PLANTED A/B" using
    `e.Site` (0=A, 1=B) for accuracy.
  - **OnBombBeginDefuse** (NEW): defuser bot calls "going for defuse" /
    "kit on me" / "im defusing cover me".
  - **OnSmokegrenadeDetonate / OnInfernoStartburn** (NEW): closest opposing
    bot calls "smoke {z}" / "molly {z}".
  - **OnPlayerBlind** (NEW): bot calls "flashed" if `BlindDuration ≥ 1.2s`.
    No zone-key dedup (it's a player-state).
  - **OnRoundTimeWarning** (NEW): one team's IGL/talker calls "TIME"
    (round-scoped, fires once per round max).

## Skipped / cut

- **Fine-grained position→zone mapping**: per advisor, building
  hardcoded site/path centroids per map is gold-plating. Used
  `PickZoneFor(map, rng)` from the existing pool everywhere; only
  `PLANTED` callout uses true site index from `EventBombPlanted.Site`.
  If realism needs improvement, future-iter could add per-map centroid
  tables.
- **HE grenade callouts** ("nade {z}"): too frequent in CS2, would
  spam over the dedup cooldown. Skipped.
- **Trade-callout follow-through into actual bot movement**: the
  "rotating" echo is chat-only. Movement-realism agent (v0.10) is
  handling movement separately.
