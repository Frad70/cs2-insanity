## What shipped

Core idea: bots get **timing realism** without any per-tick velocity writes.
The v0.6.6 drift bug was caused by velocity nudges; this version uses only
button-mask pulses (`MovementServices.Buttons.ButtonStates[0] |= bit`) and
existing `_lookUntil` / `_forceLook` plumbing.

Six features, all gated behind `_movementRealismEnabled` (menu toggle, default ON):

1. **Walk-vs-run**: Lurker / AwperPassive / Anchor / Support archetypes pulse
   `IN_SPEED` (4096) for 0.4-1.2s while moving. Roll-rate per-second:
   Lurker 0.35, AwperPassive 0.30, Anchor 0.18, Support 0.10, generic 0.08.
   Entries never creep (0%); halved while in active combat.
2. **Crouch pulse on long-range fresh target** (>800u): Awper{Passive,Aggro}
   and Anchor get a 0.4-0.8s `IN_DUCK` pulse with 65% chance. Entry skipped.
3. **Pre-aim before peek**: Lurker / AwperPassive bots, when stopped (speed<30)
   with no target, write `_forceLook` to (last-heading-yaw ┬▒ 15..40┬░, small pitch).
   Refreshes every 0.8-1.6s. Falls back to current eye-yaw if no movement
   heading recorded. Reuses the existing force-look mechanism in OnTick ŌĆö no
   parallel pipeline.
4. **Crouch-jump simulation**: rare (2% per qualifying close fresh-target while
   speed>100), max 1x per round per bot. One 33Hz tick of `IN_JUMP` then ~0.45s
   of `IN_DUCK` chained via `AddTimer(0.060f, ŌĆ”)`.
5. **Entry-fragger swing**: when an Entry archetype acquires a fresh target AND
   a teammate at <60 HP is within 700u in front of them, set
   `_aim.ReactionTimeOverrideSec[slot] = 0.080f` for that engagement (cleared
   after 1.6s).
6. **Shoulder peek**: bots with `Skill < 0.7` get a 0.2-0.3s "fake peek" where
   `_aim.ForcedTarget[slot] = (-1, until)`. The -1 sentinel means no enemy
   matches ŌåÆ goal cleared ŌåÆ bot doesn't fire. ~3%/sec roll while moving, 6s
   per-bot cooldown.

## Architecture notes

- Used a **partial class** split: `MovementRealism.cs` declares
  `public partial class InsanityRevive { ŌĆ” }` and holds the three new methods
  (`ApplyButtonPulses`, `MovementRealismTick`, `OnAimFreshTarget`). Adopted
  this because the v0.9 callouts agent was racing me on `InsanityRevive.cs`
  and every multi-line Edit failed mid-write. Smaller surface in the main file
  ŌåÆ fewer collisions.
- AimController got a callback hook `OnFreshTarget(botSlot, targetSlot, dist)`
  fired inside the existing `if (freshTarget) {ŌĆ”}` branch ŌĆö no duplicate FOV
  scan needed in the host plugin.
- AimController got a per-bot `ReactionTimeOverrideSec` dict consumed inside
  the reaction-time gate. The host clears entries when its own
  `_entryRushUntil` window expires.
- All new state dicts cleaned up in `Unload`, `OnPlayerDisconnect`, and
  per-round (`_crouchJumpUsedThisRound` reset on `OnRoundStart`).

## Spec ambiguity I resolved

The task said "if velocity < 30 and no target, set eye angles to **movement
heading** ┬▒ 30┬░". When stopped there's no live heading, so I track
`_lastHeadingYaw` whenever speed > 60 and use that. If we've never seen the
bot move (e.g. just spawned, never walked), I fall back to current eye-yaw
+ offset, which is a reasonable degenerate case (bot keeps its current angle
and gives it a small jiggle).
